בעיה 1 — Patch נכשל בשקט (סיבה לבאג WhatsApp)
מה קורה
כשהמשתמש מבקש לתקן משהו, Claude מחזיר בלוקים מהצורה:
<<<REPLACE>>>
טקסט ישן
<<<WITH>>>
טקסט חדש
<<<END>>>
הפונקציה applyHtmlPatches מנסה למצוא את "טקסט ישן" בקוד הנוכחי. אם היא לא מוצאת אותו היא עוברת הלאה בשקט (continue) — בלי להחזיר שגיאה. המשתמש רואה "עדכנתי" אבל שום דבר לא השתנה.
קובץ לתיקון
artifacts/api-server/src/services/ai/code-extractor.ts
תיקון מדויק
לפני (שורות 28-37):
typescriptfor (const { from, to } of patches) {
  if (!result.includes(from)) {
    const normalized = from.replace(/[ \t]+/g, " ").trim();
    const normResult = result.replace(/[ \t]+/g, " ");
    if (!normResult.includes(normalized)) continue;  // ← הבעיה!
    result = result.replace(
      new RegExp(normalized.replace(/[.*+?^${}()|[\]\\]/g, "\\$&")),
      to,
    );
  } else {
    result = result.replace(from, to);
  }
}
return result !== baseHtml ? result : null;
אחרי:
typescriptlet patchFailed = false;

for (const { from, to } of patches) {
  if (!result.includes(from)) {
    const normalized = from.replace(/[ \t]+/g, " ").trim();
    const normResult = result.replace(/[ \t]+/g, " ");
    if (!normResult.includes(normalized)) {
      patchFailed = true;  // ← סמן כישלון במקום לדלג
      continue;
    }
    result = result.replace(
      new RegExp(normalized.replace(/[.*+?^${}()|[\]\\]/g, "\\$&")),
      to,
    );
  } else {
    result = result.replace(from, to);
  }
}

// אם אף patch לא הצליח — החזר null כדי לאלץ full rebuild

return result !== baseHtml ? result : null;

בעיה 2 — כשה-patch נכשל, אין fallback לבנייה מחדש
מה קורה
כשה-patch נכשל ו-extractHtml מחזיר null, הקוד ממשיך הלאה כאילו הכל בסדר. הסנאפשוט לא מתעדכן, המשתמש רואה את הקוד הישן, ו-Claude לא מקבל שום פידבק על הכישלון.
קובץ לתיקון
artifacts/api-server/src/routes/projects/messages/ai-generation.ts
תיקון — אחרי שורה 615 (לאחר extractHtml)
מצא את הקוד:
typescriptlet extractedHtml = extractHtml(fullResponse, {
  isReactStack,
  previewHtml: project.previewHtml,
  intent: detectedIntent.intent,
});
הוסף מיד אחריו:
typescript// ── Patch fallback: אם patch נכשל, נסה full rebuild עם Sonnet ──────────────
const wasPatchIntent = detectedIntent.intent === "fix" || detectedIntent.intent === "edit";
if (!extractedHtml && wasPatchIntent && project.previewHtml) {
  try {
    const fallbackRes = await anthropic.messages.create({
      model: "claude-sonnet-4-5-20251001",
      max_tokens: 8192,
      system: `You are an expert web developer. The user wants to modify their app.
IMPORTANT: The patch format failed. You MUST return the COMPLETE updated HTML file.
Wrap it in \`\`\`html ... \`\`\``,
      messages: [
        {
          role: "user",
          content: `[CURRENT CODE]\n\`\`\`html\n${project.previewHtml}\n\`\`\`\n\nUSER REQUEST: ${body.content}\n\nReturn the COMPLETE updated HTML file.`,
        },
      ],
    });
    const fallbackText =
      fallbackRes.content[0].type === "text" ? fallbackRes.content[0].text : "";
    extractedHtml = extractHtml(fallbackText, {
      isReactStack,
      previewHtml: null,
      intent: "create",
    });
  } catch {
    /* non-critical */
  }
}

קובץ לתיקון

תיקון — בתוך בניית ה-system prompt
מצא:

  ? `You are an expert web developer making targeted edits...`
שנה את הסיום של ה-system prompt לעריכה, הוסף הבהרה:
typescriptconst baseSystemPrompt = isEditOrFix
  ? `You are an expert web developer making targeted edits to an existing project.
RESPOND IN THE SAME LANGUAGE AS THE USER (Hebrew → Hebrew, English → English).

⚠️ CRITICAL: The current code will be provided in the user message inside [CURRENT APP CODE — START/END] tags.
READ IT CAREFULLY before making any changes. Your patches must match the EXACT text in that code.

EDITING RULES (MANDATORY):
- Read the [CURRENT APP CODE] carefully before touching anything
- Change ONLY what the user explicitly asked to change
- Preserve ALL existing features, styles, content, and structure
- Never remove or restructure things that were not mentioned
- For HTML projects: use the patch format (<<<REPLACE>>>...<<<WITH>>>...<<<END>>>) for surgical changes
  - The text inside <<<REPLACE>>> must be COPIED EXACTLY from the current code — character for character
  - Include enough surrounding context (3-5 lines) to make it unique
  - If you're not 100% sure of the exact text, return the COMPLETE updated HTML instead
- For React/multi-file projects: output ONLY the changed files

[... שאר הכללים כמו שהם ...]`

בעיה 6 — היסטוריית שיחה חתוכה ל-6,000 תווים

בעיה 8 — System prompt ענק מדי
מה קורה
ה-system prompt הבסיסי כולל כללי עיצוב, CSS variables, typography scale, layout rules, card styles, hero section specs ועוד — יותר מ-4,000 tokens רק להוראות עיצוב. זה "גוזל" מקום מ-Claude לחשוב על הלוגיקה האמיתית של הבקשה.
קובץ לתיקון
artifacts/api-server/src/services/ai/prompts/index.ts
תיקון
ב-SHARED_OUTPUT_RULES, הסר או קצר את החלקים הבאים:

את כל ה-CSS variables המדויקות (שמור רק את העקרון, לא את הערכים)
את ה-TYPOGRAPHY SCALE המפורטת (שמור רק את הגדלים)
את ה-CARDS spec המפורטת
את ה-HERO SECTION spec המפורטת

לדוגמה, לפני (כ-800 תווים):
CARDS (every card must look polished):
- border-radius: var(--radius)
- background: var(--surface)
- border: 1px solid var(--border)
- box-shadow: var(--shadow-sm)
- transition: var(--transition)
- On hover: transform: translateY(-3px); box-shadow: var(--shadow-md)
אחרי (כ-100 תווים):
CARDS: rounded corners, subtle border, shadow, hover lift effect.
יעד: לצמצם את SHARED_OUTPUT_RULES מ~4,000 תווים ל~1,500 תווים.
