PHASE 1 — FULLY FIX THE CHAT STREAMING EXPERIENCE
============================================================

Target:
artifacts/app-builder/src/hooks/use-chat-stream.ts

This is still not good enough.

Current issue:
The assistant’s writing flow still feels like it may pause, stall, then dump chunks.
That creates a broken emotional feel.

Likely cause:
There is still a frontend typewriter/buffering mechanism layered on top of SSE.
This likely causes:
- visible pauses
- bursty rendering
- unnatural pacing
- abrupt final chunk delivery

Required fix:
Make text streaming feel truly live and continuous.

Preferred implementation:
- remove or drastically reduce artificial typewriter buffering for normal text responses
- render incoming text chunks directly or with extremely light throttling only
- avoid holding text in a queue long enough to create visible lag
- eliminate “freeze then dump” behavior
- eliminate aggressive end-of-stream flush jumps

Keep code preview streaming intact.
Do not break code generation preview events.

Review and improve:
- typewriterQueueRef
- typewriterShownRef
- startTypewriter
- appendStreamedText
- flushTypewriter
- chunk pacing logic
- final flush behavior

Success criteria for this phase:
- normal assistant text feels smooth and continuous
- no obvious stalls
- no burst dumping
- no abrupt final jump
- long answers remain responsive
- code streaming still works
