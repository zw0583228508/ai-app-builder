PHASE 2 — FIX ALL CRITICAL DATA FLOW BUGS
==================================================

You must fix all high-severity state and persistence bugs first.

This includes at minimum:

1. React files persistence
- When extractedReactFiles are generated, they must always be upserted into project_files.
- React output must not disappear after generation.
- The system must persist React project state just as reliably as HTML projects.

2. previewUpdated correctness
- done events must set previewUpdated=true whenever there is any meaningful new output:
  - new HTML
  - new React files
  - restored snapshot
  - agent-run final result
- frontend must reliably refetch when output changed.

3. Synchronization between projects.previewHtml and project_files/index.html
- For HTML projects, these must be kept in sync.
- Use a DB transaction where both must be updated together.
- Never leave partially updated state if the server crashes between writes.

4. Snapshot validity
- Do not create snapshots from invalid/truncated/partial HTML.
- Validate minimum quality before persisting a snapshot.
- Require at least a proper html/body closure or a defined valid threshold.

5. Agent flow persistence
- If autoFix changes the final output, the fixed result must be what gets saved.
- If issues are found but no fixedHtml is returned, do not silently keep stale output.
- Add deterministic fallback/retry behavior.

6. Restore flow refresh
- Snapshot restore must notify the frontend immediately.
- Trigger project update broadcast after restore.
- Ensure restored state is visible without waiting for polling.

PHASE 6 — REBUILD AI ORCHESTRATION CLEANLY
==================================================

The AI system is powerful but too hardcoded and too entangled.

Create a modular AI layer with:

1. Prompt registry
- Move all prompts out of huge route files.
- Each prompt must have a clear name, purpose, version, and expected output shape.

2. Intent routing layer
- Isolate intent detection.
- Make it testable.
- Avoid route-level sprawl.

3. Mode system
- Isolate mode detection and per-mode prompt strategy.
- Keep Entrepreneur / Builder / Developer / Maker logic separate and testable.

4. Output extraction layer
- Separate HTML extraction, React files extraction, patch extraction, continuation, and fallback handling.

5. Multi-agent analysis layer
- Do not run heavy multi-agent analysis on every trivial message.
- Introduce rules:
  - run on first message
  - run on long/complex messages
  - skip on small edits unless explicitly needed
- Reduce unnecessary latency.

6. Token reduction
- SHARED_LIBRARIES is too large.
- Reduce prompt bloat substantially.
- Only include the most important libraries by default.
- Move extended library data to optional context injection instead of always-on prompts.

7. Planning/spec flow
- Re-enable and stabilize guided planning/spec generation in a clean architecture.
- Support fast build vs guided build explicitly.
