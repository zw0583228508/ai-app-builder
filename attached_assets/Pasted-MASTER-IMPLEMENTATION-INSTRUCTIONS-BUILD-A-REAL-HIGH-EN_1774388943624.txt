GLOBAL RULES
Rule 1 — No fake behavior

Remove all fake success paths, simulation defaults, silent fallbacks that hide failure, and misleading “high-quality” naming when the implementation is not actually high quality.

Mandatory:
MOCK_MODE must be disabled by default in all production paths.
No endpoint may pretend to complete analysis if the analysis actually failed.
No rendering path may be called “high quality” unless it performs real high-quality processing.
No analysis result should be returned if confidence is too low without clearly marking it.
Rule 2 — Fail clearly, not silently

Replace every dangerous pattern like:

except Exception: pass
broad exception swallowing
silent null returns
hidden fallback with no audit trail

with:

structured error handling,
explicit failure reasons,
job-stage failure tracking,
detailed logs,
confidence and quality flags.
Rule 3 — Deterministic runtime

The system must become reproducible and environment-safe.

Required:
Pin Python version.
Pin Node version.
Lock dependencies exactly.
Ensure Docker-based reproducible environments.
Validate runtime dependencies at startup.
Validate model weights presence at startup.
Validate ffmpeg, torch, DB, Redis, and all critical services before accepting jobs.
Rule 4 — Accuracy over shortcuts

Do not build a single-model toy pipeline.
Build a hybrid best-of-breed analysis stack where each task is handled by the strongest suitable component, followed by fusion and theory-based correction.

PHASE 4 — BUILD A REAL CHORD INFERENCE ENGINE

The current simple template-matching approach is not sufficient.
Build a multi-engine chord inference system.

4.1 Required harmonic inputs

Your chord engine must combine these signals:

Signal A — HPCP / chroma harmonic evidence

Use beat-synchronous and bar-aware chroma/HPCP features.

Signal B — bass-informed root evidence

Use bass stem analysis to estimate likely harmonic roots and inversion hints.

Signal C — AMT-informed note reconstruction

Use note candidates from MT3 / Basic Pitch to reconstruct likely simultaneous pitch-class sets.

Signal D — melodic tension / lead-note influence

Use melodic anchor notes to refine chord function ranking.

Signal E — key / mode context

Use local key and mode context to re-rank candidate chords.

Signal F — temporal context

Use previous and next harmonic states for smoothing and progression plausibility.
