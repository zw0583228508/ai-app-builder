CORE EXPERIENCE PRINCIPLE

==================================================
PART 1 — CREATE A POWERFUL FIRST IMPRESSION (HOOK)
==================================================

The first screen must create an immediate emotional reaction.

REPLACE the hero content with something stronger.

Use structure like:

Headline (BIG, BOLD, CLEAR):
Build a real app in minutes — just by describing it

Subheadline:
No coding. No setup. Just tell me what you want.

Input placeholder (dynamic rotating examples):
- “Create a karaoke app with scoring and leaderboards”
- “Build me a SaaS for managing clients”
- “Make a viral AI tool people will share”

IMPORTANT:
Rotate these examples every 2–3 seconds with smooth fade animation.

Primary CTA:
Start building →

The hero must feel:
- clean
- powerful
- obvious
- irresistible

==================================================
PART 2 — MAKE THE INPUT FEEL INTELLIGENT
==================================================

The input is NOT just a text box.

Enhance it with:

1. Dynamic suggestions under input:
Clickable chips:
- AI startup idea
- Mobile app
- SaaS product
- Marketplace
- Dashboard

2. Auto-complete behavior:
As user types:
- suggest completions
- highlight smart phrases

3. Smart submit behavior:
On Enter:
- immediately trigger build flow
- no friction
- no confirmation dialogs

==================================================
PART 3 — ADD A “START BUILDING” TRANSITION
==================================================

After user submits:

Show a short animated state (1–2 seconds):

Examples:
- Analyzing your idea...
- Designing your product...
- Planning your system...

Use subtle animation (dots / progress / shimmer)

Then transition into chat.

IMPORTANT:
This creates emotional momentum.
Do NOT skip this.

==================================================
PART 4 — TRANSFORM THE CHAT INTO A GUIDED EXPERIENCE
==================================================

The chat MUST NOT feel like a blank conversation.

It must feel like:
A guided build session.

==================================================

FIRST MESSAGE STRUCTURE:

DO NOT give long answers.

Use this format:

1. Short excitement:
“Nice. This can be powerful.”

2. Immediate framing:
“Let’s define it properly.”

3. Guided choices:

Example:

Who is this for?
[Creators] [Businesses] [General users]

What level do you want?
[Simple MVP] [Advanced product] [Something viral]

Where should it run?
[Web] [Mobile] [Both]

==================================================

RULE:
ALWAYS provide clickable options where possible.

Reduce typing.

==================================================
PART 5 — CONTROL THE PACE (CRITICAL)
==================================================

Do NOT dump full responses.

Break the experience into steps.

Each step:
- small
- clear
- actionable

Flow example:

Step 1 → clarify idea  
Step 2 → define structure  
Step 3 → show preview  
Step 4 → refine  
Step 5 → expand  

Each step must feel like progress.

==================================================

While generating:

- Setting up backend logic...

Use progressive updates (streaming feel)

The user must feel:
The system is actively working, not waiting.
