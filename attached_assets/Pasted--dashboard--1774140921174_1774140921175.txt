העיצוב החדש עדיין לא תואם למה שביקשתי.

מהצילום רואים בבירור שהמערכת עדיין מרגישה כמו dashboard פנימי עם הרבה יותר מדי כלים גלויים,
במקום להרגיש כמו chat-centered builder assistant פשוט, ברור, ויוקרתי.

זה בדיוק מה שאני לא רוצה.

מה לא נכון כרגע:

1. יש יותר מדי כלים גלויים ב-sidebar
אני רואה רשימת "כלים" ארוכה מאוד:
סודות, מסד נתונים, אחסון, שגיאות, ביצועים, שימוש, WhatsApp, Planner, Brain, QA, עלויות, Runtime, Jobs ועוד.

זה גורם למוצר להרגיש כמו מערכת אדמין פנימית עמוסה,
ולא כמו assistant פשוט וחכם לבניית פרויקטים.

2. ה-chat לא מרגיש כמו הלב של המוצר
הוא קיים, אבל הוא לא מקבל מספיק dominance ויזואלי ומוצרי.
המשתמש עדיין מרגיש שהוא בתוך dashboard עם כלים,
לא בתוך flow אחד מרכזי של:
chat -> build/fix -> preview -> sync -> deploy

3. ה-preview לא מרגיש כמו שכבת התוצאה המרכזית
אני צריך שה-preview יהיה הרבה יותר ברור, חזק, מרכזי, ומחובר לתהליך העבודה.
כרגע הוא לא מרגיש כמו ה-result surface הראשי של המוצר.

4. GitHub ו-Deploy עדיין לא מרגישים first-class actions
הם צריכים להיות חלק מהשכבה הראשית של הממשק,
לא עוד יכולות שנבלעות בתוך מערכת עמוסה.

5. יותר מדי complexity חשופה למשתמש
אני כן רוצה שהיכולות יישארו קיימות.
אני לא רוצה למחוק יכולות.
אבל אני כן רוצה להחביא complexity.
המשתמש לא צריך לראות את כל העולם הפנימי הזה כל הזמן.

מה אני רוצה במקום זה:

1. למחוק את מבנה ה-sidebar הנוכחי
ה-sidebar הזה, במבנה ובתפיסה הנוכחית שלו, לא נכון.
אני לא רוצה sidebar עמוס בכלים.
אני רוצה או:
- בלי sidebar בכלל
או
- sidebar מינימלי מאוד, שמכיל רק ניווט הכרחי וברור

2. לבנות את כל הממשק מחדש סביב 3 דברים בלבד:
- Chat
- Preview / Current Project
- Primary actions: GitHub / Publish / Project status

3. כל היכולות המתקדמות צריכות להפוך ל-contextual או secondary
לדוגמה:
- DB מוצע רק אם צריך
- Secrets מוצע רק אם צריך
- Auth מוצע רק אם צריך
- Errors מופיע רק אם יש בעיה
- Versions מופיע רק כשצריך
- QA / Performance / Runtime לא צריכים לשבת מול המשתמש כל הזמן

4. אני רוצה שהצ'אט יהיה האזור המרכזי והברור ביותר
הוא צריך להיות הלב של המסך.
לא עוד widget בתוך dashboard.
אלא המרכז של כל המערכת.

5. אני רוצה שה-preview יהיה האזור השני הכי חשוב
הוא צריך להרגיש כמו התוצאה החיה של העבודה,
עם מצב ברור, refresh, publish state, version state.

6. אני רוצה ש-GitHub ו-Publish יהיו פעולות ראשיות ברורות
ב-header העליון או בשורת הפעולות הראשית,
ולא כחלק ממערכת כלים עמוסה.

7. המוצר צריך להרגיש כמו:
assistant אחד חכם שמבצע הכל דרך צ'אט,
ולא כמו IDE / admin dashboard / control panel עמוס.

מה אתה צריך לעשות עכשיו:

1. למחוק את מבנה ה-UI הנוכחי של sidebar + tools-heavy dashboard
2. לבנות app shell חדש ופשוט הרבה יותר
3. להעמיד את chat במרכז הראשי
4. להעמיד את preview כתוצאה הראשית השנייה
5. להבליט GitHub ו-Publish כפעולות ליבה
6. להעביר את כל היכולות המתקדמות למודל contextual / secondary / hidden-until-needed
7. לשמור את כל הכוח של המערכת, אבל לא לחשוף את כל המורכבות כל הזמן

אני לא רוצה עכשיו "עוד שיפור" על המבנה הזה.
אני רוצה replacement של המבנה הזה.

תתייחס לעיצוב הנוכחי כלא נכון.
אל תעשה עליו polish.
תבנה structure חדש.

המוצר צריך להרגיש כך:
- clean
- premium
- chat-first
- result-first
- minimal outside
- powerful inside

תתקן עכשיו את המבנה המוצרי של כל המסך בהתאם.
